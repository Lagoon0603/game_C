================================================================================
VOXEL SURVIVOR - NEON EDITION
================================================================================

大量のメカ敵を撃退し、生き残る3D・シューティングゲームです。
RaylibとC言語の標準機能のみを用い、3D空間でのベクトル演算、衝突判定、
および画面分割による2人対戦モードを実装しました。

================================================================================
コンパイル・実行方法
================================================================================
この課題には Makefile を使用しています。
ターミナルで本ディレクトリに移動し、以下のコマンドを入力してください。

1. コンパイル
   $ make

2. コンパイルして実行
   $ make run

3. 生成ファイルの削除
   $ make clean

※ 事前に Raylib がインストールされている必要があります。
※ コンパイラは clang を想定しています（Makefile内で変更可能）。

================================================================================
ゲームの遊び方
================================================================================
【シングルプレイ (NORMAL / HARD)】
    敵を倒してドロップするアイテム（水色＝経験値、緑色＝回復）を集め、
    レベルアップ・回復しながら生存を目指します。
    一定数の敵を倒すとボスが出現します。
    ボスを倒すと次のステージに進みます。

    ノーマルモードでプレイする時はタイトル画面でNを押してください。
    ハードモードでプレイする時はタイトル画面でHを押してください。
    ハードモードではレベル1から敵が空から降ってきます。

    [操作方法]
    - 移動　　　　： W / A / S / D キー（カメラの向き基準で移動）
    - 視点回転　　： Q / E キー
    - 射撃　　　　： マウス左クリック（押しっぱなしで連射）
    - エイム     ： マウスまたはトラックパッド
    - ダッシュ　　： SPACE キー または 左SHIFT（無敵時間あり）
    - ポーズ　　　： TAB キー（再開：TABキー / タイトルに戻る：R）

【対戦モード (VS 2P)】
    1つのキーボードを二人で使用する対戦モードです。
    左画面がP1、右画面がP2となっています。
    先に相手のHPを0にした方が勝利となります。

    [操作方法]
    [P1] 左画面
    - 移動：WASD / ダッシュ：SPACE / 射撃：左クリック
    - エイム：マウスまたはトラックパッド

    [P2] 右画面
    - 移動：矢印キー / ダッシュ：ENTER / 射撃：右SHIFT
    - オートエイム

    - ポーズ　　　： TAB キー（再開：TABキー / タイトルに戻る：R）

================================================================================
工夫したところ・アピールポイント
================================================================================

1. 画面分割によるPvPモードの実装 (glViewport / ScissorMode)
   Raylibの 'rlViewport' および 'BeginScissorMode' を活用し、
   1つのウィンドウ内で描画領域を物理的に分割することで、ローカルマルチプレイを実現しました。
   各プレイヤーに独立した 'Camera3D' 構造体を持たせることで、それぞれ異なる視点からの
   レンダリングを1フレーム内で行っています。

2. メモリ管理とオブジェクトプーリング
   ゲーム中の弾丸（最大300発）、敵（最大100体）、パーティクル（最大600個）の生成において、
   実行時の動的確保（malloc/free）を行わずに、固定長配列とフラグ管理による
   オブジェクトプーリング方式を使用している。
   これにより、大量のオブジェクトを描画してもフレームレートが安定し、
   メモリリークのリスクを排除した堅牢な設計となっている。

3. 3Dベクトル演算とカメラ制御
   - カメラの回転角（ラジアン）を基準として、三角関数（sin/cos）を用いることで
     プレイヤーを中心とした円周上のカメラ座標をリアルタイムに計算している。
   - 移動入力（WASD）は絶対座標ではなく、カメラの向き（Forward/Rightベクトル）を基準に動的に変換されるため、
     視点を回転させても入力に対する進行方向の変化のない直感的な操作が可能となっている。
   - 床のグリッド描画には 'fmodf' 関数を用いた剰余演算を取り入れ、
     プレイヤーが移動しても床が無限に続いているように見える視覚効果を
     低負荷で実装している。

4. ゲームデザインについて
   プレイヤーの入力に対するフィードバックを重視した。具体的には、
   - 射撃時のキャラクターとスクリーン上での反動
   - 敵撃破時のエフェクト
   - レベルアップによる攻撃力・連射速度の最適化
   これらを調整し、動かしていて爽快感のあるアクション性を実現している。
