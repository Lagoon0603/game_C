================================================================================
VOXEL SURVIVOR - NEON EDITION
================================================================================

【概要】
大量のメカ敵を撃退し、生き残る3D・シューティングゲームです。
RaylibとC言語の標準機能のみを用い、3D空間でのベクトル演算、衝突判定、
および画面分割による2人対戦モードを実装しました。

================================================================================
コンパイル・実行方法
================================================================================
この課題には Makefile を同梱しています。
ターミナルで本ディレクトリに移動し、以下のコマンドを入力してください。

1. コンパイル
   $ make

2. コンパイルして実行
   $ make run

3. 生成ファイルの削除
   $ make clean

※ 事前に Raylib がインストールされている必要があります。
※ コンパイラは clang を想定しています（Makefile内で変更可能）。

■ ゲームの遊び方
================================================================================
【シングルプレイ (NORMAL / HARD)】
敵を倒してドロップするアイテム（水色＝経験値、緑色＝回復）を集め、
レベルアップ・回復しながら生存を目指します。
一定数の敵を倒すとボスが出現します。

[操作方法]
- 移動　　　　： W / A / S / D キー（カメラの向き基準で移動）
- 視点回転　　： Q / E キー
- 射撃　　　　： マウス左クリック（押しっぱなしで連射）
- ダッシュ　　： SPACE キー または 左SHIFT（無敵時間あり）
- ポーズ　　　： TAB キー

【対戦モード (VS 2P)】
1つのキーボードをシェアして戦う画面分割対戦モードです。
先に相手のHPを0にした方が勝利となります。

[P1 操作] 左画面
- 移動：WASD / ダッシュ：SPACE / 射撃：左クリック

[P2 操作] 右画面
- 移動：矢印キー / ダッシュ：ENTER / 射撃：右SHIFT

================================================================================
■ 工夫したところ・アピールポイント（評価してほしい点）
================================================================================

1. 画面分割によるPvPモードの実装 (glViewport / ScissorMode)
   Raylibの `rlViewport` および `BeginScissorMode` を活用し、
   1つのウィンドウ内で描画領域を物理的に分割することで、ローカルマルチプレイを実現しました。
   各プレイヤーに独立した `Camera3D` 構造体を持たせ、それぞれ異なる視点からの
   レンダリングを1フレーム内で行っています。

2. メモリ管理とオブジェクトプーリング
   ゲーム中の弾丸（最大300発）、敵（最大100体）、パーティクル（最大600個）の生成において、
   実行時の動的確保（malloc/free）を一切行わず、固定長配列とフラグ管理による
   「オブジェクトプーリング」方式を採用しました。
   これにより、大量のオブジェクトを描画してもフレームレートが安定し、
   メモリリークのリスクを排除した堅牢な設計としています。

3. 3Dベクトル演算とカメラ制御
   - カメラの回転角（ラジアン）に基づき、三角関数（sin/cos）を用いて
     プレイヤーを中心とした円周上のカメラ座標をリアルタイムに計算しています。
   - 移動入力（WASD）は絶対座標ではなく、カメラの向き（Forward/Rightベクトル）
     を基準に動的に変換されるため、視点を回転させても直感的な操作が可能です。
   - 床のグリッド描画には `fmodf` 関数を用いた剰余演算を取り入れ、
     「プレイヤーが移動しても床が無限に続いているように見える」視覚効果を
     低負荷で実装しました。

4. ゲームデザインについて
   プレイヤーの入力に対するフィードバックを重視しました。
   - 射撃時の反動とスクリーンシェイク
   - 敵撃破時のヒットストップとフラッシュエフェクト
   - レベルアップによる攻撃力・連射速度のインフレ曲線
   これらを調整し、動かしていて爽快感のあるアクション性を追求しました。
